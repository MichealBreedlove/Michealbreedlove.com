Key Technical Highlights

Local AI Inference

RTX 4090 running Qwen 2.5 32B, DeepSeek Coder v2, and LLaMA 3.1 70B via Ollama — no cloud API dependencies for core inference.

SRE Pipeline

29 automation priorities: SLOs with burn rate tracking, incident management with postmortems, chaos testing, safety-gated actions.

Proxmox Cluster

3-node Proxmox cluster (Nova + Mira + Orin) for VM and container management. Centralized storage via TrueNAS on Nova.

Security Posture

OPNsense firewall, network segmentation, GitHub Actions secret scanning, credential hygiene policies, safety gates blocking risky automation.

GitOps Workflow

All configs backed up to GitHub daily. Ansible manages node state. Changes tracked, tested, and documented automatically.

Monitoring & SLOs

SLO targets (99.9% gateway, 99.5% inference), error budgets, 5 rolling windows for burn rate, auto-incidents on threshold breach.

How It's Operated

Monitoring

Custom dashboard with SLO burn rates, incident status, and node health. Automated health checks via Ansible every 6 hours.

Backups

Each node auto-commits sanitized state to GitHub daily via systemd timers (Linux) and scheduled tasks (Windows). Secret-scanned before push.

Incident Response

Auto-triggered incidents from SLO burn spikes or gatekeeper denials. Timeline tracking, evidence packs, and recruiter-grade postmortem generation.

Security Considerations

Network Segmentation

OPNsense firewall with VLAN-segmented traffic. Management, storage, and inference traffic isolated. 2.5G and 10G network segments.

Least Privilege

SSH key-only auth between nodes. Ansible runs with minimal required permissions. OpenClaw agents operate within defined safety tiers.

Secrets Hygiene

GitHub Actions CI blocks any push containing API keys, tokens, or credentials. Sanitize scripts strip secrets before commit. No .env files in repos.
